Install the LP-OS scraper extension

The merged Chrome extension scrapes TikTok Shop seller/order pages (order list, product analytics, LIVE, Partner Center) into LP-OS's own message store via POST /gelf, with behaviors role-gated by the LP-OS shell user.

1 · Get the extension

Download lp-os-extension.zip and unzip it (it contains a single lp-os-extension/ folder). Working from a checkout of lp-os? The same files live at extension/ — use that folder directly instead.

Do not install the legacy easierbycode.com/tok-scrape/chrome.zip — that is the OLD tok-scrape extension: it posts scrapes to the retired graylog-shim (they never reach LP-OS) and has no role gate.

2 · Load it (unpacked)

  1. Open chrome://extensions (paste it into the address bar — pages can't link to chrome://).
  2. Turn on Developer mode (top-right).
  3. Click Load unpacked and pick the unzipped lp-os-extension/ folder (or the repo's extension/).
  4. Open a TikTok Shop seller/order page — the scrape button appears (role-gated).

Chrome blocks .crx installs from outside the Web Store for normal profiles, so the supported path is Load unpacked (no .crx is published). Enterprise force-install is the only exception.

No extension? You can still run Samples-Import directly via the Claude browser skill (the samples-import skill) — it opens the page and drives it without the extension.
